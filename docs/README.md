# Toy
The toy programming language.

This repository is a test bed to allow me to test code or the Simple programming language. The syntax defined by the Toy language is a subset of the Simple language, but the code that defines the system is intended to be directly applicable to the Simple language. The intent is to develop them in parallel so that they add to each other as the projects progress.

Toy is more like a stripped down C than it is like some other. There are some important differences.

The Toy code
```
while {
  ; do things
}
```
Is equivalent to the C code
```
while(1) {
  // do things
}
```

The Toy code
```
if(some_expression) {
  ; do things
} else(another_expression) {
  ; do things
} else {
  ; do things
}
```
Is equivalent to the C code
```
if(some_expression) {
  // do things
} else if(another_expression) {
  // do things
} else {
  // do things
}
```

Toy is a work in progress and it is far from complete. See the notes in the docs directory for a more current status. I keep a TODO list there as well as actual design notes.

### Dependencies

To build Toy, you need at least the following packages

* GCC or Clang (see build instructions below)
  * note that clang-format is used to "make format"
* CMake
* Bison and Flex
* Doxygen (to build the docs)
* Graphviz (supports Doxygen)
* Git
* Firefox -- Optional.
* KDBG -- Optional. It's the best graphical GDB debugger that I know of.

### Build

There are several scripts that should be used to build and test. To change the compiler look at the last line in the ``.project_env`` file. Then re-create the build directory.

* $ ``./scripts/setup``
  This script will create the build directory and initialize cmake. There are other capabilities that are centered around making common tasks easier. Look at the file ``.project_env`` for more information.

  Setup adds the project to the ``$PATH`` and sets some environment vars to help development.

  To leave the development environment simply type ``exit`` to leave the sub shell.

* ``build`` and ``rebuild``
  These are simple shell script wrappers around ``make``.

* ``docs``
  Start firefox to view the documentation created by Doxygen.

* ``clean``
  Assumes that you are not in the development shell. Deletes the directories that are generated by building the project.


## Grammar

Rather than use the standard BNF or something the same, I am using a simplified PEG style grammar. It's easier to convert to a hand-written parser than BNF. Note that names in 'single-quotes' are considered to be not case-sensitive.

```

COMMENT ( ';'|'#'[^\n]* )

IDENTIFIER ( [_A-Za-z][_A-Za-z0-9]+ )

INTEGER_LIT ( 0|[1-9][0-9]+ )

REAL_LIT ( (([1-9][0-9]*)|0)?\.[0-9]*([eE][-+]?[0-9]+)? )

STRING_LIT ( \"[^\n\"]*\"|\'[^\n\']*\')

TEXT_LIT '{' .* '}'

program ( program_item+ )

program_item (
  data_declaration |
  function_definition |
  import_statement
  )

import_statement ( 'import' STRING_LIT )

data_declaration ( ('const'|'constant')? type_name IDENTIFIER )

function_definition (
  'nothing'|type_name IDENTIFIER '(' ( type_name IDENTIFIER ( ',' type_name IDENTIFIER )* )? ')' function_body
  )

type_name (
  'int'| 'integer' |
  'real' | 'float' |
  'string' |
  'bool' | 'boolean'
  )

# function body cannot be empty
function_body ( '{' ( function_body_item | function_body )+ '}' )

function_body_item (
  data_declaration |
  data_definition |
  function_reference |
  assignment |
  while_statement |
  do_statement |
  if_statement |
  inline_statement |
  return_statement
  )

data_definition ( data_declaration '=' expression )

function_reference ( IDENTIFIER '(' ( expression ( ',' expression )* )? ')' )

assignment ( IDENTIFIER '=' expression )

while_statement ( 'while' ( '(' expression? ')' )? loop_body )

do_statement ( 'do' loop_body 'while' ( '(' expression? ')' )? )

# loop body may be empty
loop_body ( '{' loop_body_item* '}' )

loop_body_item (
  function_body_item |
  'continue' |
  'break' |
  loop_body
)

if_statement ( if_clause else_clause* final_else_clause? )

if_clause ( 'if' '(' expression ')' function_body )

else_clause ( 'else' '(' expression ')' function_body )

final_else_clause ( 'else' ( '(' ')' )? function_body )

inline_statement ( 'inline' TEXT_LIT )

return_statement ( 'return' ( '(' expression? ')'  )? )

case_statement ( type_name '(' expression ')')

formatted_string ( ( TEXT_LIT | STRING_LIT ) ( '(' ( expression (',' expression )* )? ')' )? )

# using left recursion such as with packcc
expression (
  or_expr
  )

or_expr (
  or_expr ( '|' | 'or' ) and_expr
  )

and_expr (
  and_expr ( '&' | 'and' ) equality_expr
  )

equality_expr (
  equality_expr ( '==' | '!=' | 'equ' | 'nequ') magnitude_expr
  )


magnitude_expr (
  magnitude_expr ( '<' | '>' | '>=' | '<=' | 'lt' | 'gt' | 'lore' | 'gore' ) term_expr
  )

term_expr (
  term_expr ( '+' | '-' ) fact_expr
  )

fact_expr (
  fact_expr ( '*' | '/' | '%' ) unary_expr
  )

unary_expr (
  '-' primary_value |
  ( '!' | 'not' ) primary_value
  )

primary_value (
  IDENTIFIER |
  INTEGER_LIT |
  REAL_LIT |
  formatted_string |
  function_reference |
  cast_statement |
  '(' expression ')'
  )

```
